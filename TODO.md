# MiroFish 项目开发规划

## 🚨 高优先级（需要立即解决）
- [ ] 修复所有模块导入路径错误，确保服务正常启动 **(进行中)**
重构后的路径兼容问题

最近将services目录拆分为generation/graph/simulation三个子模块后，仍有部分导入路径未完全修复
可能还有隐藏的模块引用错误需要排查
- [ ] 核心长耗时接口改为异步任务（本体生成、图谱构建、模拟运行）
本体生成、图谱构建、模拟运行等长耗时任务都是同步接口，容易导致前端超时
大文件上传和大模拟场景下用户体验极差
- [ ] 增加LLM调用、数据库操作的重试机制和友好错误提示
LLM调用、Zep/Neo4j数据库操作没有重试机制，单次网络错误就会导致整个任务失败
错误提示不够友好，用户很难定位问题原因
